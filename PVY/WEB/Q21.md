# Základní technologie

Webová stránka se skládá ze tří hlavních pilířů:

| Technologie | Role | Přirovnání |
| :--- | :--- | :--- |
| **HTML** | Struktura a obsah | Kostra |
| **CSS** | Vzhled a design | Oblečení |
| **JavaScript** | Chování a interaktivita | Mozek / Pohyb |

## Propojení technologií
- **CSS:** Připojuje se v sekci `<head>` pomocí tagu `<link>`.
- **JS:** Připojuje se pomocí tagu `<script>`, ideálně na konec `<body>` (aby se nejdřív načetla struktura).

---

# Struktura HTML dokumentu

Každý validní HTML5 dokument musí mít tuto základní kostru:

```html
<!DOCTYPE html> <!-- Informuje prohlížeč, že jde o HTML5 -->
<html lang="cs"> <!-- Kořenový element, lang určuje jazyk -->
<head>
    <meta charset="UTF-8"> <!-- Kódování znaků -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Responzivita -->
    <title>Název stránky</title> <!-- Text v záložce prohlížeče -->
</head>
<body>
    <!-- Zde je veškerý viditelný obsah -->
</body>
</html>
```

# Elementy a atributy

> [!abstract] Element vs. Atribut
> **Element** je základní stavební blok (tag). **Atribut** poskytuje doplňující informace o elementu (píše se do otevíracího tagu).

### Časté atributy:
- `class` – přiřazení CSS třídy (může být na více prvcích).
- `id` – unikátní identifikátor (smí být na stránce jen jednou).
- `src` – cesta ke zdroji (u obrázků).
- `href` – cíl odkazu.

---

# Sémantické elementy

Sémantika dává kódu význam. Pomáhá vyhledávačům (SEO) a technologiím pro nevidomé.

- `<header>` – Hlavička stránky/sekce (logo, navigace).
- `<nav>` – Navigační menu.
- `<main>` – Hlavní, unikátní obsah stránky (pouze 1x na stránce).
- `<section>` – Tematická část stránky.
- `<article>` – Samostatný obsah (článek, novinka).
- `<aside>` – Vedlejší obsah (sidebar, reklama).
- `<footer>` – Zápatí (kontakty, copyright).

---

# Blokové vs. Inline elementy

| Vlastnost | Blokové (Block) | Řádkové (Inline) |
| :--- | :--- | :--- |
| **Začátek** | Vždy na novém řádku | Pokračuje v řádku |
| **Šířka** | Zabírá 100 % dostupné šířky | Jen tolik, kolik potřebuje obsah |
| **Příklady** | `<div>`, `<p>`, `<h1>`-`<h6>`, `<section>` | `<a>`, `<span>`, `<strong>`, `<img>` |

# Vývojářské nástroje (DevTools)

Každý moderní prohlížeč má **DevTools** (klávesa `F12` nebo `Ctrl+Shift+I`).

### Hlavní panely:
1. **Elements:** Práce s **DOM** (objektová reprezentace HTML v paměti). Umožňuje měnit HTML a CSS "za běhu" bez zásahu do souboru.
2. **Console:** Výpis chyb, testování JS příkazů, interakce s enginem.
3. **Sources:** Debugger – nastavení **breakpointů**, krokování kódu, sledování proměnných.
4. **Network:** Sledování HTTP komunikace. Vidíme metody (`GET`, `POST`), stavové kódy (`200 OK`, `404 Not Found`) a velikost souborů.
5. **Application:** Správa úložišť (**Local Storage**, **Cookies**, Session Storage).

# Základní pojmy a URL

- **Frontend:** Část webu u klienta (prohlížeč).
- **Backend:** Logika na serveru (databáze, PHP, Python).
- **API:** Rozhraní pro komunikaci mezi frontendem a backendem.

### Struktura URL:
`https://www.example.com:443/shop/products?id=10#details`

- `https://` – **Protokol** (zabezpečený).
- `www.` – **Subdoména**.
- `example.com` – **Doména** (2. a TLD 1. řádu).
- `:443` – **Port** (standard pro HTTPS).
- `/shop/products` – **Cesta** ke zdroji.
- `?id=10` – **Query parametry** (předávání dat).
- `#details` – **Fragment** (kotva na konkrétní místo).
