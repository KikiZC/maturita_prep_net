# Základní technologie

Webová stránka stojí na třech pilířích:

> [!abstract] HTML (HyperText Markup Language)
> - Základní jazyk pro tvorbu webových stránek.
> - Značkovací jazyk, který říká prohlížeči, **co** má zobrazit.
> - **Role:** Struktura / Kostra webu.

> [!abstract] CSS (Cascading Style Sheets)
> - Jazyk, který určuje **vzhled** stránky (barvy, písmo, rozestupy, responsivita, animace).
> - Díky němu vypadá web profesionálně.
> - **Role:** Design / Oblečení webu.

> [!abstract] JavaScript (JS)
> - Programovací jazyk, který dává webu **funkce a život**.
> - Umí reagovat na kliknutí, měnit obsah bez reloadu, validovat formuláře, dělat Frontend i Backend.
> - **Role:** Chování / Mozek a pohyb.

# Propojení technologií

### Propojení CSS
Umísťuje se do hlavičky (`<head>`) HTML souboru pomocí tagu `<link>`.
```html
<link rel="stylesheet" href="style.css">
```

### Propojení JS
Propojuje se pomocí elementu `<script>` a dává se ideálně na konec `<body>`.
```html
<script src="script.js"></script>
```

# Struktura HTML dokumentu

Každý validní HTML5 dokument musí mít tuto základní kostru:

- `<!DOCTYPE html>` – Říká prohlížeči, že jde o HTML5 dokument.
- `<html>` – Kořenový element celé stránky (atribut `lang` určuje jazyk).
- `<head>` – Obsahuje metadata (informace o stránce, které nejsou přímo vidět).
- `<meta charset="UTF-8">` – Nastavení kódování znaků.
- `<meta name="viewport">` – Zajišťuje responzivitu na mobilních zařízeních.
- `<title>` – Název stránky zobrazený v záložce prohlížeče.
- `<body>` – Obsahuje veškerý viditelný obsah stránky.

# Základní HTML elementy a atributy

### Časté elementy
- **Nadpisy:** Máme 6 úrovní, `<h1>` (největší) až `<h6>` (nejmenší).
- **Odstavec:** Definujeme tagem `<p>`.
- **Odkaz:** Element `<a>` s atributem `href` pro určení cíle.
	- `<a href="odkaz"> Text odkazu </a>`
- **Obrázek:** Element `<img>` s atributy `src` (cesta), `alt` (popis) a `width` (šířka).
	- `<img src="cesta" alt="popis" width="100">`

### Atributy
Doplňující informace uvnitř otevíracího tagu (světle modré kódy v editorech).
- `class` – Přiřazuje CSS třídu (pro stylování).
- `id` – Unikátní identifikátor prvku na stránce.
- `style` – Přímé vkládání CSS stylů (inline stylování).

# Sémantické elementy

Používají se pro logické označení částí stránky. Pomáhají přehlednosti kódu a vyhledávačům (**SEO**).

| Element | Význam |
| :--- | :--- |
| `<header>` | Horní část stránky/sekce (logo, navigace). |
| `<nav>` | Navigační menu s hlavními odkazy. |
| `<main>` | Hlavní a unikátní obsah stránky (pouze 1x na stránce). |
| `<article>` | Samostatný obsah, který dává smysl i mimo web (článek, novinka). |
| `<section>` | Tematická část stránky pro rozdělení obsahu. |
| `<aside>` | Vedlejší obsah (sidebar, reklama, související odkazy). |
| `<footer>` | Spodní část stránky (kontakty, copyright). |

**Další sémantické tagy:**
- `<figure>` – Samostatný obsah jako obrázek, graf nebo tabulka.
- `<time>` – Pro definici data nebo času.
- `<q>` – Krátká citace.

# Blokové vs. Inline elementy

| Vlastnost | Blokové (Block) | Řádkové (Inline) |
| :--- | :--- | :--- |
| **Začátek** | Vždy na novém řádku | Pokračuje v řádku |
| **Šířka** | Celá dostupná šířka (100 %) | Jen tolik, kolik potřebuje obsah |
| **Obsah** | Mohou obsahovat blokové i inline | Jen inline (neměly by obsahovat blokové) |
| **Nastavení** | Lze nastavit šířku a výšku | Šířka/výška se běžně nenastavují |
| **Příklady** | `<div>`, `<p>`, `<h1>`, `<section>`, `<article>`, `<header>`, `<footer>`, `<ul>`, `<li>`, `<form>` | `<a>`, `<strong>`, `<em>`, `<img>`, `<label>`, `<abbr>`, `<time>`, `<code>` |

# Nástroje pro vývoj

### Textové editory
- **Visual Studio Code (VS Code):** Nejpoužívanější, zvýrazňuje syntaxi, doplňuje kód (IntelliSense), rozšíření, podpora GITu.
- **Sublime Text:** Lehký, velmi rychlý, méně funkcí než VS Code.

Pokročilé prostředí s nástroji jako debugger, testování a hluboká analýza.
- **WebStorm:** Profesionální IDE zaměřené na JavaScript (velké projekty).
- **PhpStorm:** Primárně pro PHP, fullstack zaměření.

# Prohlížečové DevTools (F12)

Integrované nástroje v prohlížeči pro analýzu a ladění.

1. **Elements:** Práce s **DOM** (objektová reprezentace HTML v paměti). Umožňuje dynamicky měnit HTML/CSS "za běhu" pro testování změn.
2. **Console:** Rozhraní pro komunikaci s JS enginem. Slouží pro výpis chyb a testování výrazů.
3. **Sources:** Debugger pro ladění kódu.
	- **Breakpointy:** Pozastavení běhu programu na konkrétním místě.
	- **Krokování:** Procházení kódu řádek po řádku.
	- **Call stack:** Zásobník funkcí, které se právě vykonávají.
4. **Network:** Analýza síťové komunikace (HTTP/HTTPS, TCP/IP, API požadavky).
	- Sleduje se: URL, metoda (`GET`, `POST`), hlavičky, tělo, latence a stavové kódy.
	- **Stavové kódy:** `200` (OK), `301` (Redirect), `404` (Not Found), `500` (Server Error).
5. **Performance:** Měření výkonu, rychlosti vykreslování a hledání zpomalení.
6. **Application:** Správa úložišť (**Local Storage**, **Cookies**, **Session Storage**).

# Základní pojmy a URL

- **FrontEnd:** Část, kterou vidí uživatel (HTML, CSS, JS).
- **BackEnd:** Serverová část, logika a databáze (PHP, Python, C#).
- **HTTP/HTTPS:** Protokol pro komunikaci klient–server. HTTPS je zabezpečená verze.
- **API:** Rozhraní pro komunikaci mezi frontendem a backendem (klientem a serverem).

### Struktura URL
Příklad: `https://www.example.com:443/shop/products?id=10#details`

- `https://` – **Protokol** (způsob komunikace).
- `www.` – **Subdoména** (volitelná část).
- `example.com` – **Doména**.
- `:443` – **Port** (konkrétní komunikační kanál).
- `/shop/products` – **Cesta** ke konkrétnímu zdroji/stránce.
- `?id=10` – **Query parametry** (předávání dat serveru).
- `#details` – **Fragment** (kotva na konkrétní místo na stránce).
