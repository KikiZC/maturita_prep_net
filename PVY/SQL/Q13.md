# Práce s databází

## Vytvoření

``` sql
CREATE DATABASE skola;
```
Takto se dá vytvořit jednoduše databáze.

## Smazání

``` sql
DROP DATABASE skola;
```

> [!Danger] Tato akce je nevratná
> Jakmile se provede `DROP` tak systém vymaže celou databázi a není možné ji vrátit. Pouze lze znovu vytvořit

## Úprava

``` sql
ALTER DATABASE skola CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```
Toto například mění nastavení znakového kódování

# Práce s tabulkami

## Vytvoření

``` sql
CREATE TABLE student (
    ID INT PRIMARY KEY AUTO_INCREMENT,
    jmeno VARCHAR(50) NOT NULL,
    prijmeni VARCHAR(50) NOT NULL,
    narozeni DATE,
    trida CHAR(5)
);
```

Toto vytvoří tabulku i s její strukturou.
Takto nějak by mohla vypadat vizuálně:
![Vizuální tabulka studenta podle kódu](Student%20Table.png)

## Smazání

``` sql
DROP TABLE student
```

> [!Danger] Tato akce je nevratná
> Jakmile se provede `DROP` tak systém vymaže celou tabulku i s její strukturou a není možné ji vrátit. Pouze lze znovu vytvořit

## Úprava

``` sql 
ALTER TABLE student .....
```

Tímto lze upravit tabulku. místo ..... lze dát co chceme upravit. 
Například:

``` sql
-- přidání nového sloupce
ALTER TABLE student ADD COLUMN email VARCHAR(100);

-- změna typu nebo velikosti sloupce
ALTER TABLE student MODIFY COLUMN jmeno VARCHAR(100);

-- přejmenování sloupce
ALTER TABLE student CHANGE COLUMN trida trida_zkratka CHAR(6);

-- smazání sloupce
ALTER TABLE student DROP COLUMN narozeni;
```

# Datové typy

|    Kategorie    |                       Příklady                        |                       Popis                       |
|:---------------:|:-----------------------------------------------------:|:-------------------------------------------------:|
|   **Číselné**   | `INT`, `SMALLINT`, `DECIMAL(10,2)`, `FLOAT`, `DOUBLE` |        Ukládají čísla (celá i desetinná).         |
|   **Textové**   |      `CHAR(n)`, `VARCHAR(n)`, `TEXT`, `LONGTEXT`      |           Různě dlouhé textové řetězce.           |
| **Datum a čas** |        `DATE`, `TIME`, `DATETIME`, `TIMESTAMP`        |          Ukládají datum, čas nebo obojí.          |
|   **Logické**   |               `BOOLEAN` / `TINYINT(1)`                | True / False (v MySQL reprezentováno jako 1 / 0). |
|    **Jiné**     |                        `BLOB`                         |      Binární data (např. obrázky, soubory).       |

>[!info] **VARCHAR vs CHAR:**  
>`CHAR(n)` má pevnou délku (např. kód předmětu `INF01`),  
>`VARCHAR(n)` má proměnlivou délku (např. jména studentů).

# Indexy

Indexy zrychlují vyhledávání dat v tabulce (podobně jako rejstřík v knize).

|   Typ indexu    |                     Popis                     |      Syntaxe       |
|:---------------:|:---------------------------------------------:|:------------------:|
| **PRIMARY KEY** |  Hlavní klíč, jedinečný identifikátor řádku   | `PRIMARY KEY (id)` |
|   **UNIQUE**    |          Zaručuje jedinečnost hodnot          |  `UNIQUE (email)`  |
| **INDEX / KEY** |     Běžný index pro urychlení vyhledávání     | `INDEX (prijmeni)` |
|  **FULLTEXT**   | Umožňuje vyhledávání v textu (MySQL, MariaDB) | `FULLTEXT (text)`  |

## Příklad:

``` sql
-- vytvoření indexu
CREATE INDEX idx_prijmeni ON student(prijmeni);

-- odstranění indexu
DROP INDEX idx_prijmeni ON student;
```

>[!warning] **Pozor:**  
>Indexy zrychlují čtení dat, ale **zpomalují zápis** (INSERT, UPDATE, DELETE),  protože se musí aktualizovat i indexy.

Doporučení od ChadGPT

>[!info] **Optimalizace výkonu:**
>- Indexuj sloupce, které často používáš ve `WHERE` nebo `JOIN`.
>- Nepřidávej příliš mnoho indexů – mohou zpomalit vkládání dat.    
>- Pro unikátní hodnoty použij `UNIQUE INDEX` nebo `PRIMARY KEY`.
