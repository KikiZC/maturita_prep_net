# Kurzory (Cursors)

Kurzor je databázový objekt, který umožňuje **procházet výslednou sadu dat (řádky) jeden po druhém**. 
Zatímco běžné SQL dotazy pracují s celými množinami dat (set-based), kurzor pracuje iterativně (row-by-row).

**Kdy použít:**
- Pokud potřebujeme provést složitou logiku pro každý řádek zvlášť (např. volání procedury pro každého klienta).
- **Pozor:** Kurzory jsou náročné na výkon (režie serveru) a měly by se používat, jen když neexistuje hromadné (set-based) řešení.

---

# Životní cyklus kurzoru

Práce s kurzorem má 5 povinných fází, které musí následovat v tomto pořadí:

1. **DECLARE** – Definice kurzoru (přiřazení k SELECT dotazu).
2. **OPEN** – Otevření kurzoru a naplnění daty.
3. **FETCH** – Načtení konkrétního řádku do proměnných.
4. **CLOSE** – Zavření kurzoru (uvolnění výsledkové sady).
5. **DEALLOCATE** – Úplné odstranění definice kurzoru z paměti.

---

# Příkazy pro práci a cyklus

Pro automatické procházení všech řádků se využívá cyklus `WHILE`. Stav posledního načítání kontrolujeme pomocí systémové funkce `@@FETCH_STATUS`.

> [!note] Příklad použití v T-SQL
> ```sql
> -- 1. Definice proměnných a kurzoru
> DECLARE @Jmeno NVARCHAR(50);
> DECLARE muj_kurzor CURSOR FOR 
>     SELECT Jmeno FROM Uzivatele;
>
> -- 2. Otevření
> OPEN muj_kurzor;
>
> -- 3. První načtení (inicializace)
> FETCH NEXT FROM muj_kurzor INTO @Jmeno;
>
> -- Cyklus (pokud @@FETCH_STATUS = 0, načtení proběhlo v pořádku)
> WHILE @@FETCH_STATUS = 0
> BEGIN
>     PRINT 'Zpracovávám jméno: ' + @Jmeno;
>     
>     -- Načtení dalšího řádku
>     FETCH NEXT FROM muj_kurzor INTO @Jmeno;
> END
>
> -- 4. Zavření a 5. Smazání
> CLOSE muj_kurzor;
> DEALLOCATE muj_kurzor;
> ```

# Typy kurzorů

Podle toho, jak se kurzor chová k datům v čase:

- **Static (Statický):** Vytvoří si kopii dat v `tempdb`. Změny v původní tabulce provedené během práce kurzoru se v něm neprojeví.
- **Dynamic (Dynamický):** Odpovídá aktuálnímu stavu dat. Pokud někdo změní řádek, který kurzor ještě neprošel, kurzor změnu uvidí.
- **Forward-Only:** Nejjjednodušší/nejrychlejší. Lze se pohybovat pouze směrem vpřed (`FETCH NEXT`).
- **Scroll:** Umožňuje pohyb všemi směry (`FETCH PRIOR` - předchozí, `FETCH FIRST`, `FETCH LAST`, `FETCH ABSOLUTE n`).

# Příkazy řízení toku a rozhodovací konstrukce

Tyto příkazy umožňují větvení kódu a vytváření logiky skriptu (T-SQL).

## 1. IF ... ELSE
Základní podmínka. Pokud je blok delší než jeden příkaz, musí být v `BEGIN ... END`.
```sql
IF @Vek >= 18
    PRINT 'Dospělý';
ELSE
    PRINT 'Nezletilý';
```

## 2. BEGIN ... END

Seskupuje více SQL příkazů do jednoho logického bloku (analogie složených závorek `{}` v C#).

## 3. WHILE

Jediný cyklus v T-SQL. Může být řízen příkazy:

- **BREAK:** Okamžitě ukončí cyklus.
- **CONTINUE:** Skočí na další iteraci (začátek cyklu).

## 4. CASE

Výraz pro transformaci hodnot (podobné `switch`).

``` sql
SELECT Jmeno,
    CASE 
        WHEN Body > 90 THEN 'Výborně'
        WHEN Body > 50 THEN 'Prospěl'
        ELSE 'Neprospěl'
    END AS Hodnoceni
FROM Studenti;
```

## 5. TRY ... CATCH

Ošetření chyb. Pokud kód v bloku `TRY` vyvolá chybu, řízení přejde do bloku `CATCH`.

# Kurzory vs. Kurzory tabulky (Table Variables)

Často se u maturity plete pojmosloví.

- **Kurzor** prochází data po řádku.
- **Tabulková proměnná** (`DECLARE @Tabulka TABLE...`) slouží k uložení celé sady dat do paměti pro další hromadné zpracování. Často je lepší data nejdříve vyfiltrovat do tabulkové proměnné a až tu pak (v nutném případě) procházet kurzorem.

---

## Postup vytvoření

#### 1) Deklarujeme si proměnné se kterými budeme pracovat
```sql
DECLARE @Prom1 INT, @Prom2 NVARCHAR(255);
```

- například
	``` sql
	DECLARE @CustomerID NCHAR(5), @OrderCount INT;
	```

#### 2) Deklarujeme kurzor pro `SELECT` nad jehož daty bude pracovat
```sql
DECLARE <Název kurzoru>_cursor CURSOR FOR
SELECT <Něco> FROM <Odkud>
```

- například
	```sql
	DECLARE VIP_Customers_cursor CURSOR FOR
	SELECT CustomerID FROM Customers
	```

#### 3) Otevřeme kurzor
```sql
OPEN <Název kurzoru>
```

- například
	```sql
	OPEN VIP_Customers_cursor
	```

#### 4) Načteme si první řádek pomocí `FETCH NEXT`
```sql
FETCH NEXT FROM <Název kurzoru> INTO <Název jedné z promněnných>
```

- například
	```sql
	FETCH NEXT FROM VIP_Customers_cursor INTO @CustomerID
	```

#### 5) Postavíme `WHILE` cyklus, který zpracuje jeden řádek a na konci si načte další

```sql
WHILE @@FETCH_STATUS = 0
BEGIN
    <nějaká práce s tímto řádkem>
    FETCH NEXT FROM <Název kurzoru> INTO <Název jedné z promněnných>
END
```

- například
	```sql
	WHILE @@FETCH_STATUS = 0
	BEGIN
		SELECT @OrderCount = COUNT(*)
			FROM Orders
			WHERE CustomerID = @CustomerID
		IF @OrderCount > 10
		BEGIN
			INSERT INTO VIPCustomers
			VALUES (@CustomerID, @OrderCount)
		END
		FETCH NEXT FROM VIP_Customers_cursor INTO @CustomerID
	END
	```
    - V prvním kroku spočítám počet objednávek každého zákazníka
    - Ve druhém kontroluji zdali jich má více než 10 a pokud ano, tak ho zapíši do tabulky VIP zákazníků

#### 6) Nakonec kurzor zavřeme a dealokujeme - tím se smaže a nelze ho znovu otevřít

```sql
CLOSE <Název kurzoru>
DEALLOCATE <Název kurzoru>
```

- například
	```sql
	CLOSE VIP_Customers_cursor
	DEALLOCATE VIP_Customers_cursor
	```

## Shrnutí postupu

V každém kurzoru musí být vždy nějaké proměnné ([[#1) Deklarujeme si proměnné se kterými budeme pracovat|Krok 1]]), a vždy se musí deklarovat ([[#2) Deklarujeme kurzor pro `SELECT` nad jehož daty bude pracovat|Krok 2]]).
Kurzor se musí vždy otevřít a zavřít (například jak tomu je s BEGIN a END)([[#3) Otevřeme kurzor|Krok 3]] a [[#6) Nakonec kurzor zavřeme a dealokujeme - tím se smaže a nelze ho znovu otevřít|Krok 6]]).
A nakonec každý kurzor musí mít posunutí na další záznam ([[#4) Načteme si první řádek pomocí `FETCH NEXT`|Krok 4]]) a smyčku která projde všechny záznamy ([[#5) Postavíme `WHILE` cyklus, který zpracuje jeden řádek a na konci si načte další|Krok 5]])

### Celkový výstupní kód
```sql
DECLARE @CustomerID NCHAR(5), @OrderCount INT; -- Krok 1

DECLARE VIP_Customers_cursor CURSOR FOR -- Krok 2
SELECT CustomerID FROM Customers

OPEN VIP_Customers_cursor -- Krok 3

FETCH NEXT FROM VIP_Customers_cursor INTO @CustomerID -- Krok 4

WHILE @@FETCH_STATUS = 0 -- Krok 5
BEGIN
	SELECT @OrderCount = COUNT(*)
		FROM Orders
		WHERE CustomerID = @CustomerID
	IF @OrderCount > 10
	BEGIN
		INSERT INTO VIPCustomers
		VALUES (@CustomerID, @OrderCount)
	END
	FETCH NEXT FROM VIP_Customers_cursor INTO @CustomerID
END

CLOSE VIP_Customers_cursor -- Krok 6
DEALLOCATE VIP_Customers_cursor
```
