# Uložená procedura (Stored Procedure)

Uložená procedura je:
- **pojmenovaný blok SQL příkazů**
- uložený přímo v databázi
- spouští se příkazem `EXEC` / `EXECUTE`
- může mít **parametry**
- může vracet **výsledky**, **návratovou hodnotu** nebo **output parametry**

Používá se pro:
- zapouzdření logiky
- opakované použití
- zvýšení bezpečnosti a výkonu

---

# Systémové procedury

- Procedury dodávané databázovým systémem
- Slouží ke správě databáze, uživatelů, tabulek, indexů atd.

> [!example]- Příklady
> `sp_help`
> `sp_helpdb`
> `sp_tables`
> `sp_columns`

**Jsou uloženy v databázi `master`**
(*Logicky v systému, fyzicky jako objekty databáze*)

---

# Rozdíl mezi `sp_` a `xp_`

## sp_ (Stored Procedure)

- Klasické SQL uložené procedury
- Psány v T-SQL
- Běží uvnitř databáze

Bezpečnější a na běžné použití

> [!example] Příklad
> ```sql
> EXEC sp_help 'Tabulka'
> ```

## xp_ (Extended Procedure)

- Rozšířené procedury
- Implementované v C / C++
- Mohou komunikovat s **operačním systémem**

Rizikové a dnes už skoro nepoužívané

> [!example] Příklad
> `xp_cmdshell`
> -> umožňuje spouštět příkazy OS (bezpečnostní riziko)

# Rozdíl mezi funkcí a procedurou

| Vlastnost                | Procedura                    | Funkce                  |
| ------------------------ | ---------------------------- | ----------------------- |
| Volání                   | `EXEC`                       | v `SELECT`, `WHERE`     |
| Návratová hodnota        | ano / ne                     | **musí vracet hodnotu** |
| Výstup                   | result set, output parametry | návratová hodnota       |
| INSERT / UPDATE / DELETE | ✔ ano                        | ❌ ne                   |
| Transakce                | ✔ ano                        | ❌ ne                   |
| Použití v dotazu         | ❌ ne                        | ✔ ano                   |

> [!example] Funkce
> ``` sql
> SELECT dbo.MojeFunkce(5)
> ```

> [!example] Procedura
> ``` sql
> EXEC MojeProcedura 5
> ```

# Uživatelské procedury

- Procedura vytvořena **uživatelem** / **programátorem**
- Uložena v databázi (schema, typicky `dbo`)

> [!note] Vytvoření
> ``` sql
> CREATE PROCEDURE dbo.VypisUzivatele
>    @Id INT
>AS
>BEGIN
>    SELECT * 
>    FROM Uzivatel
>    WHERE Id = @Id
>END
> ```

> [!abstract] Spuštění
> ``` sql
> EXEC dbo.VypisUzivatele 1
> ```

## Procedura s OUTPUT parametrem

> [!note] Vytvoření
> ``` sql
> CREATE PROCEDURE dbo.PocetUzivatelu
>    @Pocet INT OUTPUT -- zde je definován výstup
>AS
>BEGIN
>    SELECT @Pocet = COUNT(*) FROM Uzivatel
>END
> ```

> [!abstract] Použití
> ``` sql
> DECLARE @x INT
> EXEC dbo.PocetUzivatelu @x OUTPUT
> SELECT @x
> ```

## Možnosti uložených procedur

- parametry (IN[^1] / OUT[^2]) lze kombinovat[^3]
- transakce (`BEGIN TRANSACTION`)
- podmínky (`IF`, `CASE`)
- cykly (`WHILE`)
- práce s daty (CRUD)
- vyšší výkon (plány dotazů)
- omezení přístupu k datům

## Omezení

- nelze je použít přímo v `SELECT`
- složitější ladění než běžný dotaz
- závislost na databázovém systému
- menší flexibilita než aplikační kód






[^1]: Vstupní parametry
	 - Nemají `OUTPUT` v zápisu

[^2]: Výstupní parametry
	- Označeny pomocí `OUTPUT`

[^3]: ``` sql
	CREATE PROCEDURE dbo.InfoUzivatele
	    @Id INT,
	    @Jmeno NVARCHAR(50) OUTPUT
	AS
	BEGIN
	    SELECT @Jmeno = Jmeno
	    FROM Uzivatel
	    WHERE Id = @Id
	END
	```
