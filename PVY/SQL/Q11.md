# Co je to databáze

Databáze je **organizovaný soubor dat**, který je uložen tak, aby se dal **efektivně vyhledávat, měnit a spravovat** pomocí databázového systému (DBMS – *Database Management System*).

---

# SQL jazyk

> [!abstract] SQL = Structured Query Language

Je to standardizovaný jazyk pro práci s relačními databázemi.

## Rozdělení SQL

|        Typ jazyka         | Zkratka |                  Popis                   |            Příklad příkazů             |
|:-------------------------:|:-------:|:----------------------------------------:|:--------------------------------------:|
|   Data Definition Lang.   |   DDL   | jazyk pro vytváření databázových objektů |       `CREATE`, `ALTER`, `DROP`        |
|  Data manipulation Lang.  |   DML   |       jazyk pro manipulaci s daty        | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
|    Data control Lang.     |   DCL   |    jazyk pro řízení přístupu k datům     |           `GRANT`, `REVOKE`            |
| Transaction control Lang. |   TCL   |        jazyk pro řízení transakcí        |   `COMMIT`, `ROLLBACK`, `SAVEPOINT`    | 

---

# Metadata
> [!abstract] Definice
> **Metadata** jsou „**data o datech**“ – tedy informace popisující strukturu databáze, nikoliv samotný obsah.

Příklad:
- název tabulky
-  název sloupce
-  typy dat
- klíče
- indexy

Uloženy jsou v tzn. **datovém slovníku** (data dictionary) uvnitř databáze

# Problematika Transakcí

> [!abstract] Transakce
>  = skupina operací, která se musí vykonat **celá nebo vůbec**.

## ACID 

| Zkratka |    Název    |                                                      Význam                                                       |
|:-------:|:-----------:|:-----------------------------------------------------------------------------------------------------------------:|
|  **A**  |  Atomicity  | Transakce musí být buď provedena celá, nebo vůbec. Pokud se některá část nezdaří, musí být zrušeny všechny změny. |
|  **C**  | Consistency |          Transakce musí splňovat standardy aby šla spustit na úplně cizí databázi z minimálními úpravami          |
|  **I**  |  Isolation  |                                         Transakce se navzájem neovlivňují                                         |
|  **D**  | Durability  |     Jakmile je transakce potvrzena (COMMIT), její změny musí být trvale uloženy i v případě výpadku systému.      |

# Redundance a duplicita

> [!abstract] Redundance
> nadbytečné ukládání informací v databázi
> -> zabírá místo, zvyšuje riziko nesouladu dat

> [!abstract] Duplicita[^2]
> výskyt stejných záznamů (např. dva stejní zákazníci)
> -> ztěžuje vyhledávání a může zkreslit výsledky

### Řešení:  
→ **Normalizace databáze** – rozdělení dat do tabulek tak, aby se snížila redundance a duplicita.



[^1]: My používáme Microsoft SQL Server

[^2]: Duplicita je Specifický případ Redundance
