# DHCP

- stejně jako ve Windows slouží k dynamickému přidělování IP adres
- v Linuxu se nejčastěji používá balíček `isc-dhcp-server` (nebo `dhcpd`)

## Instalace a spuštění
- **Instalace:** `dnf install dhcp-server` (Fedora/RHEL) nebo `apt install isc-dhcp-server` (Debian/Ubuntu)
- **Povolení po startu:** `systemctl enable dhcpd`
- **Konfigurační soubor:** `/etc/dhcp/dhcpd.conf`

# Konfigurace DHCP serveru

Základní nastavení se provádí definováním podsítě (subnetu).

## Parametry v dhcpd.conf
- **Subnet a Netmask** – definuje síť a masku, kde DHCP pracuje.
- **Range** – rozsah IP adres, které může server přidělit (pool).
- **Option routers** – výchozí brána (gateway) pro klienty.
- **Option domain-name-servers** – adresy DNS serverů.
- **Option domain-name** – název domény sítě.
- **Default-lease-time** – výchozí doba zapůjčení v sekundách (např. 600 = 10 min).
- **Max-lease-time** – maximální doba, po kterou si klient může adresu nechat.

# Statické přidělování (Rezervace)

Pokud potřebujeme, aby konkrétní počítač (např. server nebo tiskárna) dostal vždy stejnou IP adresu, použijeme rezervaci na základě MAC adresy.

> [!tip] Zápis rezervace
> ```text
> host nazev_rezervace {
>     hardware ethernet 00:11:22:33:44:55;
>     fixed-address 192.168.1.50;
> }
> ```

# Pokročilé funkce

## DDNS (Dynamic DNS)
- zajistí automatickou aktualizaci DNS záznamů
- když DHCP server přidělí adresu novému klientovi, okamžitě o tom informuje DNS server, který vytvoří příslušný záznam (A a PTR)

## DHCP na více sítích (DHCP Relay)
- pokud máme více propojených sítí oddělených routerem, DHCP broadcasty standardně neprojdou přes router
- řešením je **DHCP Relay Agent**
- agent naslouchá na rozhraní v síti bez serveru a přeposílá DHCP požadavky jako unicast přímo na IP adresu DHCP serveru v jiné síti

### Konfigurace DHCP Relay (Agenta)

Pokud DHCP server není ve stejné síti jako klienti, router mezi nimi musí požadavky přeposílat. K tomu slouží `dhcrelay`.

**1. Instalace:**
- `dnf install dhcp-relay` (případně v balíčku `dhcp-helper` u jiných distribucí)

**2. Konfigurace v `/etc/sysconfig/dhcrelay`:**
Do souboru zapíšeme rozhraní, na kterých má agent naslouchat, a IP adresu cílového DHCP serveru.

> [!example] Příklad zápisu
> `DHCRELAYARGS="-i eth0 -i eth1 -i eth2 10.5.5.10"`
> - **-i eth0, eth1, eth2** = rozhraní (síťovky) vedoucí do sítí, kde jsou klienti.
> - **10.5.5.10** = IP adresa tvého hlavního DHCP serveru.

**3. Spuštění služby:**
- `systemctl enable --now dhcrelay`

# DHCP Klient

Pro získání adresy na straně klienta se používají nástroje:
- **dhclient** – ruční vyžádání adresy (např. `dhclient eth0`)
- **NetworkManager** – automaticky spravuje DHCP přes `nmcli` nebo grafické rozhraní

> [!warning] Pozor na sekundy
> Nezapomeň, že v konfiguračním souboru se časy `lease-time` uvádějí vždy v **sekundách**, nikoliv v minutách!
