# Systém symbolických jmen v IP síti

- je způsob, jak lze v počítačových sítích přiřazovat jména k IP Adresám

## Možné způsoby:

- pomocí lokálního jmenného serveru
- pomocí DNS

## Doménový systém:

- internet je rozdělený na domény
- každá doména má své autorativní DNS severy, které znají konkrétní záznamy
- systém překládá symbolická jména na IP adresy, aby bylo možné navázat spojení v síti

### Důvod realizace:

- pro lidi je snadnější si zapamatovat jména než čísla
- jednodušší správa
- spolehlivost

### Princip funkce:

1. Uživatel zadá název (např. [www.seznam.cz](http://www.seznam.cz))
2. PC nejdříve zkontroluje, jestli IP adresu už nemá uloženou v cache nebo v souboru hosts.
3. Pokud ne, pošle dotaz na DNS server
4. DNS server se zeptá postupně:
	- Root serveru (kde najdu .cz domény?)
	- TLD serveru (kde najdu seznam.cz?)
	- jaká je IP  [www.seznam.cz](http://www.seznam.cz)?
5. Autoritativní server odpoví IP adresou
6. DNS server odpověď uloží a pošle ji počítači
7. Prohlížeč pak naváže spojení s nalezenou IP adresou

# DNS v INTERNETU

> [!info] DNS (Domain Name Server)
> je celosvětový systém, který převádí symbolická jména na IP adresy

## Struktura Domén:

- je hierarchický strom, který se čte zprava doleva, kde každá část oddělená tečkou představuje úroveň
- doménová struktura tvoří strom
- a pod ní jsou jednotlivé úrovně domén

1. ### Domény nejvyšší úrovně (TLD):
	- jsou na vrcholu hierarchie DNS
	- slouží k rozdělení internetu na tematické nebo geografické oblasti
		- .com
		- .cz

2. ### Domény druhé úrovně:
	- jsou přímo pod (TLD)
	- slouží jako hlavní identita webu nebo služby
		- google.com
		- seznam.cz

3. ### Pod domény (domény třetí a dalších úrovní):
	- jsou podřízené doméně druhé úrovně
	- vytvářejí se pro rozdělení služeb nebo oddělení částí webu
		- www.seznam.cz
		- www.google.com

# Doména

- je základní jednotka v systému DNS
- je to logické uspořádání počítačů, uživatelů, zdrojů (tiskárny, servery...), které jsou spravovány centrálně pomocí řadiče domény

## Subdoména

- je část domény, která se nachází pod hlavní (nadřazenou) doménou.  
- rozšiřuje nebo upřesňuje adresu v rámci jedné domény a často označuje konkrétní službu, oddělení nebo část webu
 - má vlastní adresu, ale stále patří do stejné domény

## Zóna

- je část doménového prostoru, která je spravována jedním správce nebo serverem a obsahuje informace o jménech a IP adresách pro danou oblast domén

# Druhy DNS Serveru

## Autoritativní

- obsahuje přesné a oficiální údaje o konkrétní doméně
- odpovědi z něj jsou považovány za platné a ověřené

## Neautoritativní

- neobsahuje vlastní zónu, odpověď získává od jiného serveru a uchovává ji dočasně
- takže poskytuje neověřenou odpověď

---

## Primární DNS

- hlavní server, který uchovává originální kopii zónových dat
- změny se zapisují právě sem
- z něj se kopírují na sekundární

## Sekundární DNS

- záložní server, který má kopii zóny z primárního serveru
- slouží pro zálohu a rozložení zátěže
- data se pravidelně synchronizují s primárním

---

## Catching-only

- nemá žádné vlastní zóny, pouze uchovává výsledky dříve získaných dotazů v cache
- urychluje DNS dotazy
- při prvním dotazu si odpověď vyžádá od jiného serveru
- uloží ji do cache a při dalším dotazu ji poskytne okamžitě

## Forwarding-only

- přeposílá všechny DNS dotazy na jiný server
- nepátrá sám po dopovědi – spoléhá na jiný DNS server
- přijme dotaz od klienta
- přepošle ho na předem nastavený DNS server
- vrátí odpověď klientovi

---

## Dopředná zóna

- překládá doménová jména na IP Adresy
- tvoří se zde A, AAAA, CNAME, MX záznamy

### Kanonický název uzlu (CNAME)

- je hlavní jméno uzlu, ke kterému se mohou vztahovat aliasy
- CNAME – umožňuje, aby více doménových jmen odkazovalo na stejný uzel
- např: [www.maturita.sps](http://www.maturita.sps) může být po aliasem maturita.sps

## Reverzní zóna (zpětná zóna)

- překládá IP adresu na doménová jméno
- opak dopředné zóny
- používá se speciální doména in-addr.arpa
- tvoří se zde PTR záznamy
- 192.0.2.1 -> host.example.com

## Záznamy v DNS
- určují jak se doménová jména překládají na IP Adresy

> [!abstract] A (address)
> překládá název domény na IPv4 adresu

> [!abstract] AAAA (address v6)
> překládá doménové jméno na IPv6 adresu

> [!abstract] CNAME (canonical name)
> alias přesměrovává domény na jinou doménu

> [!abstract] NS (name server)
> určuje autorativní DNS servery pro danou doménu

> [!abstract] SOA (start of authority)
> základní informace o zóně primární server, kontakt, časové údaje

> [!abstract] PTR (pointer)
> reverzní záznam, překládá IP adresu zpět na doménové jméno

> [!abstract] MX (mail Exchange)
> určuje který e-mailový server je zodpovědný za příjmení e-mailů pro danou doménu

# DNS bez Active Directory

- nezávislý DNS server – typicky jen poskytuje překlad jmen → IP adresy
- správa ruční – záznamy se nastavují v DNS zónách manuálně
- žádná integrace s uživateli nebo počítači Windows – přihlašování a oprávnění se řeší samostatně.
- použití: malé sítě, internetové servery, kde nepotřebujeme centrální správu.

# DNS s Active Directory

- DNS je integrovaný s AD – slouží k automatickému vyhledávání počítačů, serverů a služeb v doméně.
- dynamické aktualizace záznamů – počítače se samy registrují do DNS při přihlášení do domény.
- centrální správa – propojení s uživateli, skupinami, politikami (Group Policy).
- nutné pro doménové služby Windows – AD potřebuje DNS k lokalizaci řadičů domény, služeb (LDAP, Kerberos).
