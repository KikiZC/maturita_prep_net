# Služba FTP serveru v systému Linux

>[!question] Služba FTP serveru v systému Linux
>Instalace a konfigurace síťových služeb ftp, pasivní a aktivní ftp server; anonymní ftp server, osobní ftp servery uživatelů - zabezpečený přístup, zabezpečený přenos pomocí SSL mezi ftp serverem a ftp klientem) a ssh (instalace a konfigurace ssh, ověřování přístupu pomocí hesel či osobních certifikátů uživatelů).

---

## Definice

>[!info] FTP
>FTP = File Transfer Protocol
>Slouží k přenášení souborů po síti

>[!info] Aktivní/Pasivní FTP
>Aktivní
>
>- Server vytváří připojení ke klientovi
>
>Pasivní
>
>- Nástupce Aktivního FTP
>- Server pouze poslouchá a čeká na připojení od klienta
>- Firewall-friendly

>[!info] Anonymní FTP
>Na rozdíl od normálního FTP, které vyžaduje jméno/heslo, tak k anonymnímu FTP má přístup kdokoliv s adresou serveru

---

## Základní info

- **Šifrování**: Ne (jeho variantu můžou podporovat)
- **Porty**: 20 (přenost dat) a 21 (příkazy)
- **Transportní protokol**: TCP

---

## Instalace a konfigurace

### Instalace

```bash
dnf install vsftpd -y
```

### Po instalaci

>[!warning] Povolení FTP
>V základu firewall a SELinux **blokuje** FTP, aby tedy fungovalo tak se musí v těchto službách povolit a nebo vypnout
>(Protože Linux, tak vypnutí je v našem případě a prostředí spolehlivější)
>
>**Povolení**
>- Selinux
>	- `setsebool -P ftpd_use_passive_mode=on`
>	- `setsebool -P ftpd_anon_write=on`
>- Firewall
>	- `firewall-cmd --add-service=ftp --permanent`
>
>**Vypnutí**
>- Selinux
>	- `nano /etc/selinux/config`
>	- ```bash
>	...
>	SELINUX=enforcing => SELINUX=permissive
>	...
>- Firewall
>	- `systemctl stop firewalld`
>	- `systemctl disable firewalld`

### Zkouška spojení

Nejjednodušší cesta jak vyzkoušet spojení s FTP serverem je samozřejmě se na něj připojit. Dosáhneme toho pomocí druhé stanice, která bude plnit roli FTP client (pomocí FileZilla nebo průzkumníku souborů).

Abychom se nezdržovali učty tak použijeme možnost anonymního připojení. V základu je vypnuté a zapneme ho zakomentováním následujícího řádku v `/etc/vsftpd/vsftpd.conf`

```bash
...
anonymous_enable=NO => #anonymous_enable=NO
...
```

Alternativně lze `NO` změnit na `YES`.

>[!warning] Restartuj službu
>
>Po změně konfigurace vždy restartuj službu
>
>```bash
>systemctl restart vsftpd.service
>```

Teď přejdeme na připojení a to z CLI na druhé stanici pomocí příkazu `ftp`. Pokud ho nemáme nainstalovaný tak to uděláme následovně:

```bash
dnf install ftp -y
```

Připojení se provede následovně:

```bash
ftp -a <adresa FTP serveru>
```

`ftp` - oznamujeme, že se chceme připojit k FTP serveru
`-a` - anonymní připojení
`<adresa FTP serveru>` - IP adresa/DNS serveru s FTP

Úspěch potom vypadá následovně:

```bash
root@fedora:~ ftp -a <adresa server>
Connected to <adresa serveru>
...
```

### Konfigurace

Kořenová složka pro veškerou konfigurace se nachází v `/etc/vsftpd` a obsahuje hlavní konfigurační soubor je `vsftpd.conf`.

#### Často používané parametry

Níže jsou popsány jedny z nejdůležitějších parametrů, se kterými se můžeme setkat v konfiguračním souboru

```bash
# Anonymní přihlášení (tj. bez jména/hesla)
anonymous_enable=YES/NO

# Přihlášení k FTP pomocí lokálních (na serveru) účtů
local_enable=YES/NO

# Povolí zápisu do FileSystemu (smazat, uložit, mkdir, rmdir,...)
write_enable=YES/NO

# Povolí anon. už. nahrávání souborů
# MUSÍ být nastavený "write_enable=YES"
anon_upload_enable=YES/NO

# Povolí anon. už. vytvářet složky
# MUSÍ být nastavený "write_enable=YES"
anon_mkdir_write_enable=YES/NO

# Místní uživ., ke kterému se příhlásí anon. uživ.
ftp_username=ftp

# Pokud YES, tak při anon. nahrávání nastaví na souborech
# jako "owner" uživatele uvedeného v "chown_username"
chown_uploads=YES/NO

# Uživatel který se nastaví jako "owner" u anon.
# nahraných soborů
# MUSÍ být nastavený "chown_uploads=YES"
chown_username=<USERNAME>

# Pokud YES, tak při přihlášení místního uživatele
# bude uživatel umístěn do své domovské složky
chroot_local_user=YES/NO

# Pokud je "chroot_local_user=NO", tak tento parametr
# nastavuje, kteří uživatelé BUDOU přesunutý při přihlášení
# do domovského adresáře.
#
# Pokud je "chroot_local_user=YES", tak tento parametr
# nastavuje, kteří uživatelé NEBUDOU přesunutý při přihlášení
# do domovského adresáře.
chroot_list_enable=YES/NO

# Lokace souboru s uživateli pro předchozí 2 paramtery
chroot_list_file=/etc/vsftpd/chroot_list

# Nastavuje, jestli FTP má poslouchat pouze na IPv4 adrese
# Pokud "listen=YES", tak musí být "listen_ipv6=NO"
listen=YES/NO

# Nastavuje, jestli FTP má poslouchat na IPv4 a IPv6 adrese
# Pokud "listen_ipv6=YES", tak musí být "listen=NO"
listen_ipv6=YES/NO

# Načítá uživ. jména z "userlist_file" a podle
# "userlist_deny" zakazuje/povoluje přihlášení
userlist_enable=YES/NO

# Pokud NO, tak pouze uživ. v "userlist_file" mají
# povolené přihlášení
# Pokud YES, tak uživ. v "userlist_file" mají
# zakázané přihlášení
# MUSÍ být nastavený "userlist_enable=YES"
userlist_deny=YES/NO

# Lokace souboru s uživateli pro předchozí 2 paramtery
userlist_file=/etc/vsftpd/userlist

# Služba ověřování uživatelských jmen/hesel
# Pokud nevíš, neměn
pam_service_name=vsftpd
```

#### Min. config pro použití místních uživatelů

Minimální paramtry, potřebné pro nastavení FTP tak, aby bralo pouze přihlášení místních uživatelů

```bash
anonymous_enable=NO

local_enable=YES

write_enable=YES

listen=NO

listen_ipv6=YES

userlist_enable=YES
# Defaultně soubor /etc/vsftpd/user_list

pam_service_name=vsftpd
```

Uživatele se defaultně přihlásí do své domovské složky

#### Min. config pro použití anon. uživatelů

>[!warning] Pozor
>Anon. přístup nevyžaduje heslo => možná bezpečnostní hrozba

Minimální paramtry, potřebné pro nastavení FTP tak, aby bralo pouze přihlášení místních uživatelů

```bash
anonymous_enable=YES

local_enable=NO

write_enable=YES

anon_upload_enable=YES

anon_mkdir_write_enable=YES

ftp_username=ftp

listen=NO

listen_ipv6=YES

userlist_enable=YES
# Defaultně soubor /etc/vsftpd/user_list

pam_service_name=vsftpd
```

Defaultně se anon. uživ. připojují do složky `/var/ftp`
Tato složka obsahuje v základu jenom složku `pub`, které nastavíme práva `777` a zároveň jako vlastníka nastavíme uživatele `ftp`

---

Konfigurace lze samozřejmě nakombinovat, nebo do nich přidat spoustu dalších parametrů které jsou dostupné v `man vsftpd.conf`

## Zabezpečený přenos

### SFTP

SFTP = SSH File Transfer Protocol (port 22)
Jak název napovídá, tato varianta FTP využívá SSH, které při komunikaci využívá asymetrické šifrování => přenášená data nejsou v plain text podobě

SFTP je nezávislé na `vsftpd` a je stažené společně s balíčkem `openssh`

```bash
dnf install openssh -y

# nebo

dnf install sftp -y
```

Pro připojení následně použijeme příkaz `sftp` v následující podobě

```bash
sftp <uzivatel>@<adresa serveru>
```

Při prvním připojení budete dotázáni na správnost klíče a pokud potvrdíte tak zadáte heslo uživatele a můžete stahovat/nahrávat

>[!warning] Pozor
>K `root` se defaultně nelze přihlásit
>Pro povolení je potřeba odkomentovat řádak v `/etc/ssh/sshd_conf`
>
>```bash
>PermitRootLogin prohibit-password
>
># prohibit-password zabraňuje použití hesla pro ověření
># pokud se chceme přihlásit heslem, je potřeba přepsat
># prohibit-password > yes
>```
>
>a potom `ssh` restartovat

>[!pros] Výhoda
>SSH podporuje ověřování pomocí veřejného klíče uživatele, který když přidáte na server, tak vám odpadá nutnost stále zadávat heslo

---

### FTPS

FTPS = File Transfer Protocol Secured (port 989-data, 990-přikazy)
Na rozdíl od SFTP, FTPS používá pro šifrování a ověřování SSL certifikát

Pro povolení SSL připojení k serveru potřebujeme přidat do `/etc/vsftpd/vsftpd.conf` následující parametry

```bash
# Povolí využití SSL certifikátu k šifrování
ssl_enabled=YES

# Nastaví cestu k souboru s certifikátem
rsa_cert_file=<cesta k soubor>

# Nastaví cestu k souboru s privátním klíčem
rsa_private_key=<cesta k soubor>
```
