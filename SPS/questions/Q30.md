# Služba FTP serveru v systému Linux

>[!question] Služba FTP serveru v systému Linux
>Instalace a konfigurace síťových služeb ftp, pasivní a aktivní ftp server; anonymní ftp server, osobní ftp servery uživatelů - zabezpečený přístup, zabezpečený přenos pomocí SSL mezi ftp serverem a ftp klientem) a ssh (instalace a konfigurace ssh, ověřování přístupu pomocí hesel či osobních certifikátů uživatelů).

---

## Definice

>[!info] FTP
>FTP = File Transfer Protocol
>Slouží k přenášení souborů po síti

>[!info] Aktivní/Pasivní FTP
>Aktivní
>
>- Server vytváří připojení ke klientovi
>
>Pasivní
>
>- Nástupce Aktivního FTP
>- Server pouze poslouchá a čeká na připojení od klienta
>- Firewall-friendly

>[!info] Anonymní FTP
>Na rozdíl od normálního FTP, které vyžaduje jméno/heslo, tak k anonymnímu FTP má přístup kdokoliv s adresou serveru

---

## Základní info

- **Šifrování**: Ne (jeho variantu můžou podporovat)
- **Porty**: 20 (přenost dat) a 21 (příkazy)
- **Transportní protokol**: TCP

## Instalace a konfigurace

### Krok 0

```bash
dnf install vsftpd -y
```

### Krok 0.5

>[!warning] Povolení FTP
>V základu firewall a SELinux **blokuje** FTP, aby tedy fungovalo tak se musí v těchto službách povolit a nebo vypnout
>(Protože Linux, tak vypnutí je v našem případě a prostředí spolehlivější)
>
>**Povolení**
>- Selinux
>	- `setsebool -P ftpd_use_passive_mode=on`
>	- `setsebool -P ftpd_anon_write=on`
>- Firewall
>	- `firewall-cmd --add-service=ftp --permanent`
>
>**Vypnutí**
>- Selinux
>	- `nano /etc/selinux/config`
>	- ```bash
>	...
>	SELINUX=enforcing => SELINUX=permissive
>	...
>- Firewall
>	- `systemctl stop firewalld`
>	- `systemctl disable firewalld`

---

### Krok 1

Konfigurace se nachází `/etc/vsftpd`

#### Zkouška spojení (anon)

Nejjednodušší cesta jak vyzkoušet spojení s FTP serverem je samozřejmě se na něj připojit. Dosáhneme toho pomocí druhé stanice, která bude plnit roli FTP client (pomocí FileZilla nebo průzkumníku souborů).

Abychom se nezdržovali učty tak použijeme možnost anonymního připojení. V základu je vypnuté a zapneme ho zakomentováním následujícího řádku v `/etc/vsftpd/vsftpd.conf`

```bash
anonymous_enable=NO => #anonymous_enable=NO
```

Alternativně lze `NO` změnit na `YES`.

>[!warning] Restartuj službu
>
>Po změně konfigurace vždy restartuj službu
>
>```bash
>systemctl restart vsftpd.service
>```

Teď přejdeme na připojení a to z CLI na druhé stanici pomocí příkazu `ftp`. Pokud ho nemáme nainstalovaný tak to uděláme následovně:

```bash
dnf install ftp -y
```

Připojení se provede následovně:

```bash
ftp -a <adresa FTP serveru>
```

`ftp` - oznamujeme, že se chceme připojit k FTP serveru
`-a` - anonymní připojení
`<adresa FTP serveru>` - IP adresa/DNS serveru s FTP

Úspěch potom vypadá následovně:

```bash
root@fedora:~ ftp -a <adresa server>
Connected to <adresa serveru>
...
```
