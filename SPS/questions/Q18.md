# Procesy a spouštění programů v systému Linux

## Definice

>[!info] Program
>Soubor obsahující spustitelný kód

>[!info] Proces
>Instance programu/spuštěný program
>
>Každý proces má ID = PID
>Program může mít více procesů/instancí

---

## Proces - spouštění, ukončení, pozastavení, popředí, pozadí

Každý příkaz (`dnf`, `nano`,...) v Linuxu spouští proces, tedy instanci programu. `dnf` a jiné příkazy jsou v backendu `alias`, které odkazují na spustitelný soubor (soubor s možností `x` - `execute`)

Představme si 2 ilustrační programy, který budou každou sekundu vypisovat slovo

```bash
# program1
# /root/ahoj

while true
do
	echo ahoj
	sleep 1
done
```

```bash
# program2
# /root/nazdar

while true
do
	echo nazdar
	sleep 1
done
```

*Předpokládáme, že soubor má již nastavený pravá `x`*

### Spouštění

Pro jednoduché spuštění `program1` bez `alias` použijeme následující příkaz

```bash
/root/ahoj
```

Tímto jsme spustili příkaz v popředí, čili v aktuální instanci terminálu (`tty`) nemůžeme dělat nic jiného.

---

### (Po)Zastavení

Abychom mohli opět používat aktuální `tty`, tak máme 2 možnosti, jak program přerušit

1. Pozastavení - `CTRL` + `Z`
	- Proces se dočasně přeruší
	- Pomocí příkazu `jobs` uvidíme jeho stav a ID
2. Ukončení - `CTRL` + `C`
	- Proces se ukončí trvale

Pro obnovení pozastaveného procesu použijem příkaz

```bash
fg %[ID]
```

Tedy pokud jsme pozastavili jen jeden proces tak napíšeme

```bash
fg %1
```

---

### Popředí, Pozadí

Tímto se dostáváme ke spouštění procesů v popředí a pozadí

Bylo již zmíněno, jak se spustí příkaz v popředí (viz [[#Spouštění]]). Abychom spustili příkaz na pozadí, použijem příkaz

```bash
/root/ahoj &
```

Zásadní rozdíl mezi procesem v popředí a pozadí je, že když je proces v pozadí, tak lze používat aktuální `tty`, narozdíl od procesu v popředí

Pokud má proces output v `tty`, tak bude akorát náročnější napsat souvislejší string, protože proces bude vizuálně přepisovat terminál

*Př: příkaz `jobs` s procesem v pozadí a outputem v `tty`*

![[/SPS/imgs/Q18/background-proc.png]]

Pokud je proces spuštěn v pozadí a chceme ho přenést do popředí popužijeme tento příkaz (předpoklad - beží pouze jeden proces)

```bash
fg %1
```

Pokud je příkaz v popředí a chceme ho přenést do pozadí, učiníme buď ukončením (`CTRL`+`C`) a potom spuštěním na pozadí, nebo pozastavením (`CTRL`+`Z`) a potom příkazem

(Předpoklad - běží pouze jeden proces)
```bash
bg %1
```

### Sekvenční spouštění

#### `;`

*Př:*
```bash
/root/ahoj ; /root/nazdar
```

Projdeme si postupně co příkaz dělá

1. Spustí se program `/root/ahoj`
2. `;` čeká než, se program `/root/ahoj` ukončí aby spustil další příkaz
	- `/root/ahoj` je infinite loop, takže ho budeme muset ukončit ručně `CTRL`+`C`
3. Spustí se program `/root/nazdar`

---

#### `&&`

*Př:*
```bash
true && echo ahoj
-> ahoj

false && echo ahoj
->
```

`&&` hlídá, jestli příkaz před ním vrátí Success (true) nebo Error (false).
Pokud vrátí Success, tak provede příkaz po něm.
Pokud vrátí Error, tak příkaz po něm se neprovede.

---

#### `||`

*Př:*
```bash
true || echo ahoj
->

false || echo ahoj
-> ahoj
```

`||` je přesně opačná situace od `&&`, kdy zde to provede další příkaz pokud předcházející příkaz vrátil Error místo Success

---

### Paralelní spouštení

Paralelní spouštění znamená, že najednou spustíme více procesů => z logiky věci musí běžet minimálně *počet procesů - 1* v pozadí

Učiníme tak následovně

```bash
/root/ahoj & /root/nazdar    # pouze program1 běží v pozadí

/root/ahoj & /root/nazdar &  # oba programy běží v pozadí
```

## Parent-child procesy

Vztah typu parent-child mezi procesy vzniká ve chvíli, kdy již existující proces použije tzv. `fork()` system call, který spůstí 2+ podprocesy.

>[!example] Příklad
>Kopírujeme soubor ze složky A do složky B a tento přenos je zabalen do jednoho procesu.
>Po procesu chci, aby přenesl soubor a zároveň aby ukazoval progress bar.
>Rodičovský proces tedy zavolá `fork()` a ten vytvoří dva potomky:
>- Potomek1 - přenos dat
>- Potomek2 - progress bar
>
>V moment co skončí přenos dat, tak se ukončí potomek1 i potomek2 a následně rodičovský proces
>
>*Protože progress bar a přenos dat jsou na sobě zavislý, nedává smysl aby pro toto existovali dva normalní procesy, ale je lepší přesně tento případ kdy 2 podprocesy jsou zastřešeny pod jedním hlavním (viz [článek](https://www.geeksforgeeks.org/dsa/difference-between-process-parent-process-and-child-process/))*
