# Služba webového serveru Apache v systému Linux

>[!question] Služba webového serveru Apache v systému Linux
>Web server Apache ( globální nastavení, nastavení defaultního serveru a konfigurace virtuálních serverů na https protokolu, virtuální hosting, Open SSL).

---

## Definice

>[!info] OpenSSL
>OpenSSL je open-source knihovna, která řeší kryptografii pro SSL/TLS traffic

## O Apache

Apache je jedna ze dvou hlavních služeb používaných na Linuxu pro hostování webových stránek. (Druhá je NGINX)

Jedná se o Open-Source službu s jednoduchou konfigurací a flexibilitou co se týká konfigurace.

---

## Instalace a spuštění

Ačkoliv se službě říká Apache, tak balíček se jmenuje `httpd`. Pro instalaci tedy použijem následující příkaz:

```bash
dnf install httpd
```

V základu je `httpd` vypnuté, takže pro zapnutí a nastavení aby se `httpd` spouštělo hned po bootu zadáme tento příkaz:

```bash
 systemctl enable --now httpd
```

Pokud na portu 80 neběží žádný jiný proces tak se služba zapne, v opačném případě se nám příkaz vrátí s chybou. Pro ověření stavu služby slouží tento příkaz:

```bash
systemctl status httpd
```

>[!warning] Firewall
>`firewalld.service` defaultně blokuje HTTP traffic
>Pro načtení stránky z jiné stanice je nutno přidat pravidlo, nebo vypnout firewall. Nejjednoduší způsob jak přidat HTTP do firewallu je následovně
>```bash
>firewall-cmd --add-service=http --permanent
>```

## Konfigurace

Cesta k hlavnímu konfiguračním souboru je následující:

```bash
/etc/httpd/conf/httpd.conf
```

---

### `Listen`

První důležité řádky konfigurace jsou

```bash
#Listen 12.34.56.78:80
Listen 80
```

První řádek pužijem v případě, že na serveru je více IP adres (NIC) a chceme zajistit, aby byly zpracovány HTTP requesty pouze z konkretní adresy (defaultně zakomentováno)

Druhý řadek říká, aby `httpd` zpracovalo všechny requesty, které přijdou na port 80, nehledě na IP adresu (NIC)

---

### `ServerName`

Dále máme v defaultu zakomentovaný `ServerName`. Díky této konfiguraci určujeme na jaký DNS název má `httpd` reagovat. Pokud server nemá DNS záznam tak sem zadáme IP adresu

---

### `DocumentRoot`

`DocumentRoot` slouží k určení složky, kde se nachází soubory pro webové stránky -> kam si `httpd` sáhne když dostane request, defaultně

```bash
DocumentRoot /var/www/html
```

---

### `<Directory>`

Když víme, kam si `httpd` sáhne když dostane request, tak musíme vědět, že `httpd.conf` omezuje přístup ke složkám, resp. file systemu

Ještě před `DocumentRoot` je tento blok kódu

```bash
<Directory />
	AllowOverride none
	Require all denied
</directory>
```

`<Directory />` určuje k jaké složce se mění přístup, zde `/`
`AllowOverride` určuje, jestli může být tento blok kódu upraven pomocí `.htaccess` souboru, které jsou v adresáři webové stránky
`Require` určuje, kdo (ne)má přístup

Dohromady vyčteme, že do složky `/` nemá nikdo přístup a žadný `.htaccess` soubor nemůže toto nastavení přepsat

Z toho plyne, že musí být nastavený přístup do složky `/var/www/html`, když je nastavena jako `DocumentRoot` a to následovně

```bash
<Directory /var/www/html>
	AllowOverride none
	Require all granted
</Directory>
```

---

### `<VirtualHost>`

`<VirtualHost>` je po nastavení globální konfigurace nejdůležitější vlastnost v celém `httpd.conf`. Pokud je na serveru s jednou IP hostováno více webových stránek je nutné zajistit aby každý request skončil ve správném cíli -> úkol `<VirtualHost>`

*Příklad - na serveru jsou stránky **web1.cz** a **web2.cz***

```bash
<VirtualHost *:80>
	ServerName web1.cz
	DocumentRoot /var/www/web1
</VirtualHost>

<VirtualHost *:80>
	ServerName web2.cz
	DocumentRoot /var/www/web2
</VirtualHost>
```

Oba `<VirtualHost>` tu poslouchají na portu 80 a čtou si HTTP header, kde hledaj cílovou URL. Podle cílové URL se rozhodují, jestli uživatele poslat do složky web1 nebo web2

---

### SSL/TLS

HTTP traffic není šifrovaný a je přenášen v plain-text formátu = každý, kdo odchytí packety s HTTP trafficem je může přečíst.

Aby se tomu zabránilo, používa se HTTPS, které používá se certifikát k ověření a de/šifrování client-server komunikace.

Apache pro SSL/TLS komunikaci používá `ssl_module`, pro instalaci je třeba použít následujcí příkaz

```
dnf install mod_ssl
```

následně můžeme upravit adekvátně config

*Pro účely kódu se bere v potaz přítomnost certifikátu (fiktivní případ)*

```bash
LoadModule ssl_module modules/mod_ssl.so

<VirtualHost *:443>
	ServerName web3.cz
	SSLEngine on
	SSLCertificateFile    "/cert/web3.cz/web3.cz.cer"
	SSLCertificateKeyFile "/cert/web3.cz/web3.cz.key"
</VirtualHost>
```
